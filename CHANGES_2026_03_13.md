# Changes Report - March 13, 2026

## Objective
1. Add a note plugin in the social tab.
2. Implement credential saving for authentication.
3. Provide an app function explanation and design/maintenance plan.

## Implemented Changes

### 1. Social Tab: Journal/Notes Plugin
- **SocialBlock**: Increased `totalTabs` to 4 to accommodate the new "Journal" tab.
- **SocialPage**: 
    - Updated `TabController` length to 4.
    - Added `SocialNotesDashboard` as the 4th tab.
    - Added a new action in the bottom action bar (Floating Action Icon) for the 4th tab to allow creating new social journal entries directly.
    - Implemented localization for the new tab title ("Journal").
- **SocialNotesDashboard**: 
    - Refactored from a standalone page to a tab-compatible widget by removing `Scaffold`, `AppBar`, and `SwipeablePage`.
    - Integrated with `ProjectNoteDAO` to watch notes specifically in the 'social' category.
- **Localization**: Added `journal` and `social_notes` keys to `app_en.arb` and `app_vi.arb`.

### 2. Security: Credential Saving
- **AuthBlock**: Updated multiple authentication flows to securely persist credentials using `SecureStorageService`:
    - **Password Login**: Now explicitly saves credentials upon successful login.
    - **Passkey Login**: Saves the username and sets a placeholder password for biometric re-authentication.
    - **Google Sign-In**: Saves user metadata for credential persistence.
    - **Registration**: Automatically saves credentials after a successful account creation.
- **SecureStorageService**: Confirmed usage of `flutter_secure_storage` for encrypted local storage of sensitive data.

### 3. App Function & Design Strategy
- **Functionality**: Ice Gate is a "Life Orchestration Engine" that gamifies productivity, health, and social interactions. It transforms daily tasks and metrics into "XP" and "Points" to encourage a balanced lifestyle.
- **Design Plan**:
    - **Architecture**: Follows a layered architecture (Data -> Orchestration -> UI) using `signals` for reactive state management.
    - **Scalability**: New features are implemented as "plugins" or "internal widgets" to keep the core lean.
- **Maintenance Plan**:
    - Use `ReactiveBlock` for state logic to keep UI widgets stateless and easy to test.
    - Maintain strict separation between local (Drift/SQLite) and remote (Supabase) data synchronization.
    - Regularly update localization files to support the bilingual user base.
