# CHANGES 2026-04-12

## Health Steps Desktop Guard Fix

### Problem
macOS app was polling HealthKit and writing `steps=0` to `health_metrics`, corrupting the real iPhone data synced via PowerSync.

### Root Cause
`_startHealthSync()` in `DataLayer.dart` ran on all platforms including desktop. macOS has no step sensor so HealthKit returns 0. The existing guard (`if isDesktop && steps==0 && calories==0`) was too weak — it passed when calories were non-zero.

### Fix
Added a hard platform gate at the top of `_startHealthSync()` that skips the entire HealthKit sync timer on desktop platforms (macOS/Linux/Windows/Web). Desktop now only receives health data passively through PowerSync from the phone.

### Files Changed
- `lib/initial_layer/DataLayer.dart` — Added desktop platform guard in `_startHealthSync()`

### Impact
- Desktop apps no longer write sensor data — UI still displays correct values from PowerSync
- Manual health entries (water, weight, exercise) continue to work on all platforms
