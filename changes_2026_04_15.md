# Changes Report - 2026-04-15 (ICE Gate)

## Summary
Migrated the Social Blocker from the Settings page to an Internal Plugin architecture.

## Key Insights
- **Internal Widget System**: The application uses a central registration system in `RoutingLayer.dart` to populate the `internal_widgets` database table.
- **Navigation Flow**: Decoupling settings from core features improves discoverability by placing tools (like the App Blocker) in the "Canvas" or "Plugin Grid" on the Home page.
- **Idempotency**: Modified the registration logic to check for existing entries before inserting, ensuring stability across app restarts.

- [x] **RoutingLayer.dart**: Added "Social Blocker" to the initialization list. Points to `/social/blocker`.
